Свойства компонента

Components ››

Вы можете изменить любой компонент, будь то в библиотеке или на схеме. Если вы хотите произвести базовые настройки компонента, лучше отредактировать соответствующий компонент в библиотеке. Это, так сказать, оригинал, т.е. все компоненты, которые вы перетаскиваете отсюда на чертеж, имеют одинаковые свойства. И наоборот, если вы хотите изменить только один компонент один раз, лучше отредактировать соответствующий компонент на чертеже. Это оставляет нетронутым "оригинал" в библиотеке.

 

Чтобы изменить компонент, вызовите его свойства.

 

·          На чертеже достаточно выделить компонент щелчком мыши.

·          Чтобы отредактировать компонент библиотеки, дважды щелкните компонент в библиотеке. Компонент теперь выделен цветом как «выбранный».

 

Свойства компонента теперь отображаются на панели свойств в правом углу окна:

 

Идентификатор

Идентификатор - это идентификационная буква компонента, за которой следует серийный номер (например, «R1»). Вы можете ввести этот идентификатор вручную для каждого компонента или использовать очень полезную опцию автоматической перенумерации.

 

видимый

С помощью этой опции вы можете показать или скрыть идентификатор компонента. Эта опция может быть полезна для некоторых компонентов, у которых вообще нет идентификатора или которые, возможно, вообще не являются компонентами.

 

автоматическая нумерация

С помощью этой опции вы полностью оставляете нумерацию программе. В качестве идентификатора введите, например, только букву «R» (без порядкового номера!). Затем программное обеспечение рассматривает все компоненты на чертеже, которые также имеют идентификатор «R» (и также активировали опцию автоматической перенумерации), как принадлежащие друг другу, и автоматически прикрепляет последовательные номера к идентификатору.

 

При удалении компонентов на чертеже в автоматической нумерации компонентов появляются пробелы. Чтобы закрыть это снова, вы можете в любой момент изменить нумерацию всего чертежа в программе. Для этого выберите команду «Перенумеровать компоненты ...» в меню «Компоненты» или нажмите соответствующую кнопку на панели инструментов.

 

Ценить

Значение представляет собой либо числовое значение компонента (например, «1k»), либо обозначение типа (например, «BC547» или «SN7400»). С помощью значения запроса параметра у вас есть возможность ввести значение непосредственно при перетаскивании компонента на чертеж.

 

видимый

С помощью этой опции вы можете показать или скрыть значение детали. Эта опция может быть полезна для некоторых компонентов, которые не представляют ценности или, возможно, вообще не являются компонентами.

 

значение запроса

Если вы активируете эту опцию, каждый раз, когда вы перетаскиваете этот компонент из библиотеки на чертеж, появляется небольшое диалоговое окно, в котором вы можете ввести соответствующее значение. С помощью этой опции вы можете заставить себя, например, сразу ввести значения всех резисторов. Если вы этого не сделаете, вам нужно будет добавить все значения не позднее, чем по завершении рисования. Тогда значение можно быстро забыть.

 

Добавить в список компонентов

Если вы отмените выбор этого поля, компонент не будет учитываться при создании списка компонентов. Это полезно, например, если компонент на самом деле вообще не является компонентом.

 

Дополнительные тексты 1..4 для списка компонентов

Эти дополнительные тексты появляются только в списке компонентов. Здесь вы можете ввести особую информацию, такую ​​как дизайн, материалы или номер заказа.

 

 

Описание библиотеки

При необходимости вы можете ввести здесь текст, который затем будет отображаться в библиотеке непосредственно под компонентом. Обратите внимание, что этот текст не должен быть слишком длинным.

 

 

Редактор компонентов

Используйте кнопку редактора компонентов, чтобы открыть редактор компонентов для соответствующего компонента. Вы можете редактировать внешний вид компонента по своему усмотрению в редакторе компонентов.

 

 

 

Компонент-Контакты

Компонент может иметь любое количество контакты или, еще лучше, описания контактов. Это специальные имена внутри компонента. Преимущество перед фиксированными текстами заключается в том, что вы можете быстро и легко изменить эти тексты в диалоговом окне компонента. Таким образом, вам не нужно сначала просматривать редактор компонентов.
